If you're new to {{siteName}}, here's the short version. Three things worth knowing — none of them surprising, all of them useful.

## 1. The blog is part of the site, not a separate thing

This blog (`/blog/`) sits inside the main {{siteName}} site. It uses the same domain, the same navigation, the same theme. Posts here aren't on a different platform pretending to be {{siteName}} — they're {{siteName}} content with a publish date and a topic.

Practical implication: when you bookmark a post or share a link, it's a real {{siteName}} URL. No tracking redirect, no third-party host.

## 2. Categories and tags do different jobs

- **Category** is the broad bucket — there's only one per post. Think "what section of the site does this belong to?"
- **Tags** are cross-cutting — multiple per post. Think "what topics or themes does this touch?"

If you're browsing, the **category archive** pages (linked from individual posts) give you the topical streams. The **tag archive** pages (also linked from posts) give you everything that mentions a specific subject across categories. Both are valid ways to discover content.

## 3. Search engines see structured data

Each post emits machine-readable structured data behind the scenes:
- Article schema (title, dates, author, body)
- Breadcrumb schema (Home → Blog → this post)
- FAQ schema (when the post has FAQ items, like this one)

That doesn't change anything for human readers, but it means search engines understand the content properly — including showing FAQ results in some queries directly in the search page. It's invisible plumbing that compounds over time.
